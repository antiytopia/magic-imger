# Magic Imger

SEO image pipeline tool на `Node.js` + `sharp`.

Проект не позиционируется как очередной "компрессор картинок". Идея в другом: это компактный, быстрый и понятный инструмент для обработки изображений в пайплайне, который умеет конвертацию форматов, ресайз и оптимизацию как по одному файлу, так и пакетно по целой папке.

## Идея проекта

Сделать инструмент с минимальным количеством кода и максимальной практической пользой:

- один понятный движок обработки изображений;
- три основные функции без расползания фичей;
- удобный CLI для Linux и macOS;
- Windows-режим с UI-окном как тонкая оболочка над тем же ядром;
- читаемая архитектура, которую другой разработчик сможет быстро подхватить.

## Что делает продукт

Ровно 3 основные функции:

1. `resize`
2. `compress`
3. `convert`

И 2 обязательные мелочи, без которых инструмент будет неудобным:

1. `batch mode` для обработки папки
2. аккуратный `output flow`: сохранять имена файлов и складывать результат в отдельную папку

## Основные сценарии

### 1. Resize

- вписать изображение в заданные размеры;
- сохранить пропорции;
- опционально сделать `center crop`, если нужен exact size.

Примеры задач:

- подготовить превью 400x400;
- сделать карточки товаров одинакового размера;
- уменьшить изображения для сайта без ручной возни.

### 2. Compress

- с `quality`, если формат поддерживает потерянное сжатие;
- без `quality`, когда нужна lossless или "умная" оптимизация без явного ухудшения там, где это возможно.

Примеры задач:

- уменьшить вес JPG/WebP/AVIF;
- прогнать PNG через аккуратную оптимизацию;
- быстро получить более легкие ассеты для веба.

### 3. Format Conversion

Поддерживаемые форматы MVP:

- `jpg`
- `png`
- `webp`
- `avif`

Примеры задач:

- конвертировать папку `png` в `webp`;
- собрать `avif` для production;
- перегнать `webp` обратно в `png` для внутренних процессов.

## Batch mode

Пакетная обработка обязательна с первого дня.

Минимальные требования:

- обработка всей папки;
- сохранение исходных имен;
- отдельная выходная папка;
- предсказуемая структура результата без хаоса в корне проекта.

Пример:

```bash
magic-imger convert ./input --to webp --output ./dist
magic-imger resize ./input --width 1200 --height 1200 --fit contain --output ./dist
magic-imger compress ./input --quality 82 --output ./dist
```

## Почему `Node.js` + `sharp`

Это лучший прагматичный стек для такого MVP:

- очень мало кода;
- высокая скорость;
- `sharp` уже умеет resize, quality и format conversion;
- отлично подходит для пакетной обработки;
- удобно делать CLI без лишних слоев;
- под капотом у `sharp` взрослый и проверенный стек, а не игрушечное решение.

## Кроссплатформенность

### Linux

Основной режим: CLI через терминал.

### macOS

Рекомендуемый режим для MVP: тоже CLI через терминал.

Причина простая: вы не сможете нормально тестировать отдельный macOS GUI, а поддержка UI на платформе без реального тестирования быстро превратится в источник багов. Для MVP лучше держать один надежный CLI.

### Windows

Требуемый режим: UI-окно.

Рекомендация по архитектуре:

- не делать отдельную бизнес-логику для UI;
- вынести всю обработку в общий CLI/core;
- Windows UI сделать тонкой оболочкой, которая просто собирает параметры и вызывает тот же движок.

Так получится сохранить минимальный код и не дублировать логику между платформами.

## Принципы проекта

- минимальный код;
- максимум удобства;
- предсказуемое поведение;
- хорошая читаемость для других разработчиков;
- без перегруза настройками в MVP;
- сначала полезный pipeline, потом расширения.

## Предлагаемый MVP

### Команды

- `resize`
- `compress`
- `convert`

### Общие аргументы

- `--input`
- `--output`
- `--recursive` при необходимости
- `--overwrite` только как явная опция

### Аргументы для `resize`

- `--width`
- `--height`
- `--fit contain|cover`
- `--crop center`

### Аргументы для `compress`

- `--quality`
- `--lossless` где поддерживается

### Аргументы для `convert`

- `--to jpg|png|webp|avif`
- `--quality` при необходимости

## Что не нужно тащить в MVP

- сложный редактор изображений;
- десятки фильтров;
- ручную поканальную настройку кодеков;
- полноценный кроссплатформенный GUI на всех ОС сразу;
- попытку поддержать "все форматы мира".

## Позиционирование

Это не "еще один image compressor".

Это свой компактный SEO image pipeline tool:

- для подготовки изображений под сайт;
- для пакетной генерации веб-форматов;
- для предсказуемой оптимизации ассетов;
- для понятного использования разработчиками и контент-командами.

## Следующий шаг

После утверждения документации можно переходить к реализации MVP:

1. собрать CLI на `Node.js`;
2. реализовать три команды на базе `sharp`;
3. добавить пакетную обработку папки;
4. затем поверх этого сделать Windows UI как thin wrapper.
